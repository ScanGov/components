Buttons

Two button patterns in regular use: primary and outline. Both get custom scangov styling on top of Bootstrap. Be careful with btn-outline-primary (Bootstrap class, customized) vs btn-primary-outline (scangov-only, for nav-pills — documented separately below).

Primary

Standard call-to-action button. Uses var(--bs-primary) for background and border, so it inherits the site's brand color. Add an icon with me-2 margin before the label.

Outline

Use for secondary actions. Non-obvious behavior: scangov.css adds margin-top: 1rem to .btn.btn-outline-primary automatically unless .btn-lg is also present. This is intentional spacing for inline use alongside body text — but wrap in a flex/grid container with align-items-center or override with mt-0 if the automatic margin causes layout issues.

btn-primary-outline — nav-pills only

a.btn-primary-outline is not a general button variant. Despite the similar name, it's a different class from Bootstrap's btn-outline-primary. It's styled specifically for .nav-pills a.btn-primary-outline — the active/hover/focus rules in scangov.css are all scoped to .nav-pills. Using it outside a nav-pills context will produce unstyled or inconsistent output. See nav-tabs.html for correct usage in context.
