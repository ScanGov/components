ScanGov

Forms

Standard Bootstrap 5 form classes apply throughout. scangov.css adds minor hover/focus enhancements to .form-control — no extra classes needed for that. Always pair inputs with <label> elements using matching for/id attributes.

Text inputs

Use .form-control for text, email, password, and textarea inputs. Wrap each input and label in .mb-3 for consistent spacing. 

<div class="mb-3">
  <label for="input-email" class="form-label">Email address</label>
  <input type="email" class="form-control" id="input-email">
</div>

Search

Add shadow to the search input — this is the established pattern across ScanGov sites. Use with a <datalist> for autocomplete or connect to a search handler via JS. 

<input type="search" class="form-control shadow" id="search"
  placeholder="Search…" aria-label="Search">

Select

Use .form-select for dropdowns. 

<label for="input-select" class="form-label">Indicator</label>
<select class="form-select" id="input-select">
  <option selected>All indicators</option>
  <option>Accessibility</option>
</select>

Checkboxes

Use .form-check wrappers with .form-check-input and .form-check-label. Each checkbox needs its own unique id. 

<div class="form-check mb-2">
  <input class="form-check-input" type="checkbox" id="check-1">
  <label class="form-check-label" for="check-1">Label</label>
</div>

File upload

<label for="input-file" class="form-label">Upload domain list</label>
<input class="form-control" type="file" id="input-file" accept=".csv,.txt">

Validation states

Use Bootstrap's .is-valid / .is-invalid on the input and a sibling .valid-feedback / .invalid-feedback element. Add novalidate on the form to manage validation via JS.

Looks good.
Enter a valid domain, e.g. agency.gov.
<input type="text" class="form-control is-valid" id="...">
<div class="valid-feedback">Looks good.</div>

<input type="text" class="form-control is-invalid" id="...">
<div class="invalid-feedback">Error message.</div>
