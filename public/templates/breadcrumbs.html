Breadcrumbs

Two distinct breadcrumb patterns are in use across ScanGov sites — choose based on context, not preference. Both live in components/_includes/.

Default breadcrumb

Use for standard page-level navigation (docs, standards, features, any non-profile page). Uses Bootstrap's standard .breadcrumb with .breadcrumb-default and .font-monospace styling. Source: _includes/breadcrumb-default.html.

  1. Features
  2. Sites dashboard
<nav aria-label="Breadcrumb" class="breadcrumb-default mb-3">
  <ol class="breadcrumb small font-monospace">
    <li class="breadcrumb-item"><a href="/parent/">Parent</a></li>
    <li class="breadcrumb-item active" aria-current="page">Current page</li>
  </ol>
</nav>

Profile breadcrumb

Use for the org → domain hierarchy in the site-monitoring context. Custom classes .bc-org, .bc-domain, and .bc-sep give the separator and each link type distinct theme-aware colors (defined in scangov.css, lines ~535–590). A single <li> holds the full path — don't nest multiple <li> elements. Source: _includes/breadcrumb-profile.html.

  1. State of Iowa / iowa.gov 
<nav aria-label="breadcrumb" class="fs-5 font-monospace">
  <ol class="breadcrumb mb-0">
    <li class="breadcrumb-item">
      <i class="fa-solid fa-hand-point-right" aria-hidden="true"></i>
      <a href="/org/org-name/" class="bc-org">Org Name</a>
      <span class="bc-sep"> / </span>
      <a href="/profile/domain-gov/overall/" class="bc-domain">domain.gov</a>
    </li>
  </ol>
</nav>

Profile breadcrumb — org current

When the org itself is the current page (not a specific domain), use .bc-org-current on the org link instead of .bc-org and omit the separator and domain link.

  1. State of Iowa 
<li class="breadcrumb-item" aria-current="page">
  <i class="fa-solid fa-hand-point-right" aria-hidden="true"></i>
  <a href="/org/org-name/" class="bc-org-current">Org Name</a>
</li>
